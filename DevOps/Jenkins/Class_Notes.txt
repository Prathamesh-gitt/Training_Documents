	- Jenkins - Build Orchestration Tool :::
	
		- Is an open-source build orchestration Tool 
		- Used to Create the end-to-end CICD Pipeline to automation application builds and deployments
		- Used to perform any scheduled process automation 
	
	- Jenkins Architecture :: CI & CD 
	
		- Jenkins is based on master/Controller & slave/Node
		
			- Jenkins Master/Controller (VM) :		
								- Install Jenkins 
								- To Create Jenkins Jobs/Projects/Pipelines and scheduled to run on the Jenkins Slave Nodes 
		
				- Jenkins Slave_Node (VM) (Build Server):	
								- Install the Build Tools 
								- To Execute the Jobs  

	Alternative to Jenkins :
	
		- GitHub-Actions 
		- gitlab-ci 
		- Bamboo
		- Azure Pipelines 
		- AWS Code Pipelines 
	
	- Setup Jenkins Architecture :::

	Scenario1 :	Application Build and Release frequency is very less.
	
		Jenkins_Master (VM)

	- Create the Jenkins Master & Slave Node Configurations :::
	
	
	Setup:
	
		- Jenkins Master:(VM)
				==> To Create Jenkins CI/CD Pipeline Projects and schedule to the builds in the slave_nodes
				==> Tools:
						- git 
						- jdk
						- jenkins

			- Jenkins Slave Node:(VM)
				==> Perform Application Build for Java Maven Applications
				==>	Tools:
						- git 
						- jdk
						- Maven 
						
						
		- Setup the SSH Connection:
		
			- On Slave Node - Create linux user, and ssh keys.
			
		- Login to Jenkins Console -> Setup the Jenkins Node Configurations.
			
Create Jenkins Pipeline:

pipeline {
    agent { label 'slave1' }
    
    environment {
        JAVA_HOME = '/usr/lib/jvm/java-17-openjdk-amd64'
        PATH = "${JAVA_HOME}/bin:${env.PATH}"
    } 
    
    stages {
        stage('SCM_Checkout') {
            steps {
                echo 'Perform SCM Checkout for Java Application'
				git 'https://github.com/PL-DevOps-Cloud-2705/java-webapp-project.git'
            }
        }
        stage('Application Build') {
            steps {
                echo 'Perform Java Application Build'
				sh 'mvn clean package'
            }
        }
        stage('Deploy to QA Server') {
            steps {
                echo 'Deploy to QA Server'
            }
        }
    }
}
